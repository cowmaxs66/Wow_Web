# v1.17.0 发布说明 - 命令执行回执与审计可视化

## 版本定位
v1.17.0 面向远程操作闭环：Server 不再只能看到“命令已写入队列”，Web Admin 可以看到 Client 执行后的成功或失败回执。

## 新增能力
- 新增共享协议 `ClientCommandReceiptRequest`、`ClientCommandReceipt` 和 `ClientCommandReceiptList`。
- Management Server 新增 `POST /api/client/command-receipts/{client_id}`，用于 Client 上报命令执行结果。
- Management Server 新增 `GET /api/client/command-receipts/{client_id}`，用于 Web Admin 查询最近回执。
- Server 为每台 Client 保留最近 100 条命令执行回执。
- Client monitor 执行远程白名单命令后，会回传成功或失败摘要。
- Web Admin 远程操作页新增“最近执行回执”，显示当前单台 Client 最近执行结果。

## 安全边界
- 回执接口只接受现有白名单命令类型。
- 回执摘要最长 2000 字符。
- 回执当前只保存在 Server 内存中，Server 重启后丢失。
- 本版本仍未提供登录鉴权、操作员身份、审批、持久化审计和防重放。
- 本版本不包含大漠二进制、授权文件、私有脚本、真实账号和 JSONL 运行数据。
