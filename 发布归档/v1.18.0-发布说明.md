# v1.18.0 发布说明 - 工程化地基修补

## 版本定位
v1.18.0 不新增用户功能，重点修补工程化地基：共享远程命令清单、Client 命令错误类型、Server app 拆分和 GitHub Actions CI。

## 新增能力
- 新增共享远程命令常量与 `REMOTE_COMMAND_TYPES`。
- 新增 `is_supported_remote_command`，Server 白名单和 Client 执行分支共用同一份命令类型清单。
- Client 远程命令执行返回 `RemoteCommandError`，区分不支持命令和执行失败。
- `management-server/src/app.rs` 拆分为：
  - `app.rs`：路由、handler、上线日志。
  - `app_validation.rs`：API 请求校验。
  - `app_tests.rs`：app 路由测试。
- 新增 GitHub Actions CI：
  - Rust fmt。
  - Rust clippy。
  - Rust workspace test。
  - Web Admin `npm ci`。
  - Web Admin build。

## 安全边界
- CI 不编译 Delphi DmBridge，因为 GitHub runner 没有 RAD Studio 和本机大漠私有环境。
- CI 不生成正式发布包，不上传 Release 资产。
- 本版本只是命令清单单一来源的过渡方案，尚未引入完整命令 registry。
- 本版本不包含大漠二进制、授权文件、私有脚本、真实账号和 JSONL 运行数据。
