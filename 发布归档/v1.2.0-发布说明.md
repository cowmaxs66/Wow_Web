## 本版已包含
- Management Server 保存每个 Client 最近 50 条状态历史。
- 新增 `GET /api/client/history/{client_id}` 历史查询 API。
- 新增 `ClientStatusHistory` 共享响应结构。
- Web Admin 新增历史趋势面板，展示样本数、在线样本、脚本切换、最近样本和最近记录。
- Web Admin 历史趋势线只基于真实上报样本绘制，不补假数据。

## 本版未包含
- SQLite、Postgres 或其他生产数据库。
- Server 重启后的历史恢复。
- 长期趋势、聚合报表、留存统计和告警。
- 账号体系、鉴权、TLS、日志流、脚本下发和命令执行。

## 验证摘要
- `cargo fmt --all --check` 通过。
- `cargo clippy --workspace -- -D warnings` 通过。
- `cargo test --workspace` 通过，client-agent 21 项、management-server 8 项、shared-types 3 项。
- `npm run build` 通过。
- P8 Server/Client 烟测通过，连续三次真实上报后历史 API 返回 `history_total = 3`、`history_limit = 50`。
- Playwright fallback 桌面 1440x920 与移动 390x844 通过，无横向溢出。

## 注意事项
P8 历史存储是 Management Server 进程内短期历史，不是生产持久化数据库。进入生产趋势分析前必须先设计数据库 schema、索引、保留策略、清理任务和备份恢复。
