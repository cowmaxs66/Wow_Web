# v1.22.0 发布说明 - 多机器管理与通讯效率优化

## 发布状态

- 状态：本地编译包已生成，GitHub Release 未创建。
- 目标：完成 P29/P30，多机器分组标签、批量操作安全边界、monitor jitter 和 `/api/client/sync` 合并同步。

## 新增能力

- Client 状态新增 `identity.display_name`、`identity.group` 和 `identity.tags`。
- Client 本机设置窗口新增显示名、分组和标签输入。
- `config.apply` 支持远程修改显示名、分组和标签，但不允许修改 `client.id`。
- Web Admin Client 列表、详情和快照分析展示分组与标签。
- Web Admin 对“全部客户端”批量消息和批量命令增加显式确认。
- Client monitor 增加 jitter，默认最大 1500ms。
- Server 新增 `POST /api/client/sync`，一次请求完成状态上报、消息返回和命令返回。
- Client monitor 优先使用 sync，失败时回退旧状态上报、消息拉取和命令拉取链路。

## 边界

- 本版本仍未加入登录鉴权、角色权限、审批流和长期审计数据库。
- 批量确认只防误触，不替代生产权限控制。
- 编译包不包含 `dm.dll`、`RegDll.dll`、大漠授权文件、私有脚本、账号资料、JSONL 历史数据。
- 当前远端 GitHub Release 仍停留在 v1.19.0。
