# v1.23.0 发布说明 - Client 分页过滤与审计持久化

## 发布状态

- 状态：本地编译包已生成，GitHub Release 已创建。
- 目标：完成 P31，补齐多机器管理的 Server 端分页过滤、分组/标签查询、操作审计 JSONL 和 Web 审计面板。

## 新增能力

- 共享协议新增 `ClientStatusPage`，用于返回分页元数据和当前页 Client 状态。
- Server 新增 `GET /api/client/status-page`，支持 `page`、`page_size`、`group`、`tag`、`online` 和 `search` 查询。
- 共享协议新增 `ServerAuditEvent` 与 `ServerAuditEventList`。
- Server 新增 `GET /api/server/audit`，可查询最近消息、命令和命令回执审计事件。
- Server 新增 `MANAGEMENT_SERVER_AUDIT_PATH`，配置后审计事件追加写入 JSONL，并在启动时回放最近事件。
- Web Admin Client 列表改为 Server 端筛选与分页，保留旧 `/api/client/status` 回退。
- Web Admin 远程操作页新增审计面板，展示最近 Server 操作与 Client 回执摘要。

## 边界

- 本版本仍未加入登录鉴权、角色权限、审批流、操作者身份和数据库审计。
- 审计 JSONL 默认不启用；启用后仍属于本机运行数据，不得提交 GitHub。
- 分页查询基于 Server 当前内存快照和历史回放恢复，不是数据库索引查询。
- 编译包不包含 `dm.dll`、`RegDll.dll`、大漠授权文件、私有脚本、账号资料和 JSONL 运行数据。
