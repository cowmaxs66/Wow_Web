# v1.24.0 发布说明 - DM 正式包与多选客户端操作

## 发布状态

- 状态：本地编译包已生成，GitHub Release 已创建。
- 目标：完成 P32，让 Client 包默认允许 DM 脚本权限，总包和 Client 分包携带 DM 运行 DLL，并让 Web 远程操作支持勾选多台 Client。

## 新增能力

- Client 默认配置 `script_security.allowed_permissions` 已包含 `dm.access`。
- Client 默认配置测试已覆盖 `dm.access`，防止后续误删 DM 权限。
- Web Admin 远程操作目标改为 Client 列表框，可通过复选框选择单台或多台 Client。
- Web Admin 远程配置默认勾选 DM 权限，DmBridge 路径默认指向包内 `dm-bridge/Win32/DmBridge.dll`。
- 发布脚本从 `I:\图色\工具\大漠\7.2149` 或 `WOW_DM_RUNTIME_DIR` 复制 `dm.dll` 与 `RegDll.dll`。
- 总包和 Client 分包包含 `dm-bridge/Win32/DmBridge.dll`、`dm-bridge/Win32/dm.dll`、`dm-bridge/Win32/RegDll.dll`。
- Server 分包保持不包含 DM 运行 DLL，避免把 Client 侧插件文件混入 Server-only 部署包。

## 边界

- `dm.dll` 与 `RegDll.dll` 只进入 ignored 的发布包目录，不提交到 Git 源码仓库。
- 发布包携带 DLL 不等于目标机器已完成 COM 注册或授权；DM 实机使用仍需要确认目标机器授权状态。
- 本版本不包含授权文件、账号资料、商业脚本、CHM/CHW 文档、JSONL 运行数据、签名私钥和 `.env`。
- 公开分发前必须确认大漠插件授权与再分发边界；如授权不允许公开分发，应改用私有 Release 或让用户本机安装 DM 运行库。
- 本版本仍未加入登录鉴权、角色权限、审批流和操作者身份，生产联网前必须补齐。
