# v1.25.0 发布说明

## 版本定位
`v1.25.0` 是 Lua 热推送与内部测试模式版本，重点解决实机测试时修改 Lua 后 manifest hash 不匹配，以及管理端无法直接向选中 Client 推送脚本的问题。

## 已包含
- Client 默认进入内部测试模式：
  - `lua.enabled = true`
  - `script_security.enabled = false`
  - `allowed_permissions = ["host.log", "config.read", "dm.access"]`
- Web Admin 远程操作页新增 Lua 热推送面板。
- Web Admin 可对勾选 Client 下发：
  - `script.deploy_bundle`
  - `script.start`
  - `script.stop`
  - `script.status`
  - 原有 `script.run_bootstrap`
- Client 新增热推送写入器，把 Server 下发的 Lua 内容写入 `scripts/` 目录。
- Client 新增 Lua 软停止：Lua 停止后 monitor 仍在线，可继续接收 Server 命令。
- Server 新增 `script.deploy_bundle` payload 校验，内部测试模式允许无 manifest 推送。
- Web Admin 增加 15 秒重复下发拦截，避免状态延迟下重复点击造成重复命令。

## 原问题说明
v1.24.0 用户下载包中的 `scripts/bootstrap.lua` 被修改后，`scripts/bootstrap.manifest.json` 仍保留旧 SHA-256。Client 启用安全校验时会拒绝执行，这是预期行为，不是包损坏。

v1.25.0 默认关闭 `script_security`，用于内部测试直接修改和热推送 Lua。后续需要重新开启安全门时，可在后台设置中打开，并同时推送 manifest。

## 使用方式
1. 启动 Server 分包或总包中的 `management-server.exe`。
2. 启动 Client 分包或总包中的 `client-agent.exe`。
3. 在 Web Admin 左侧进入 `远程操作`。
4. 勾选目标 Client。
5. 在 Lua 热推送面板填写 Lua 内容。
6. 保持默认内部测试模式，点击推送。
7. 如需立即执行，勾选“推送后立即执行”。
8. 如需暂停脚本，点击 `停止 Lua`；Client 仍会在线。

## 边界
- 热推送只写入 Client 包内 `scripts/` 目录。
- `script.stop` 是软停止，不强杀正在执行中的 Lua。
- 总包和 Client 分包继续携带 `DmBridge.dll`、`dm.dll` 和 `RegDll.dll`。
- Server 分包不包含 Client、脚本、DmBridge 或 DM 运行 DLL。
- 源码仓库不提交授权文件、账号资料、签名私钥、真实商业脚本和 JSONL 运行数据。
