# v1.26.0 发布说明

## 版本定位
`v1.26.0` 是 DM 绑定诊断与桌面控制台版本，重点处理 `dm_bridge_bind_window` Access violation 诊断不足、Server 默认打开浏览器标签页、Client 日志查看不够直观，以及 Lua 常用接口不完整的问题。

## 已包含
- DmBridge `BindWindow` 前置校验：
  - `hwnd > 0`
  - `IsWindow(hwnd)`
  - `display/mouse/keypad` 非空
- DmBridge 绑定失败时输出：
  - `hwnd`
  - `display`
  - `mouse`
  - `keypad`
  - `mode`
  - `last_dm_error`
- `FindWindow` 找不到窗口时输出 class/title。
- Lua 新增：
  - `dm.find_window_required`
  - `dm.bind_window_try`
  - `dm.safe_bind_window`
  - `dm.with_bound_window`
  - `dm.get_color_rgb`
  - `dm.wait_color`
  - `dm.sleep_ms`
  - `dm.now_ms`
- Web Admin DM/Lua 示例改用 `dm.safe_bind_window`。
- Server 托盘默认打开 Edge App 模式桌面控制台，不再打开普通浏览器标签页。
- Client 新增 `--log-window`，托盘菜单新增“查看日志窗口”。
- 新增 `技术设计/Lua接口表与使用说明.md`。

## BindWindow 建议
实机测试时，先使用：

```lua
local ok, err = dm.safe_bind_window(hwnd, "normal", "windows", "windows", 0)
```

如果 `normal` 可用，再尝试 `gdi` 或 `dx`。`dx` 模式由大漠内部决定是否兼容当前窗口和桌面环境，框架只能增强诊断，不能保证所有窗口都能绑定。

## 边界
- `dm.safe_bind_window` 只避免 Lua 脚本直接中断，不代表绑定一定成功。
- Server 桌面控制台依赖目标机器安装 Microsoft Edge；缺少 Edge 时回退默认浏览器。
- Client 日志窗口只读取本机 `logs/client-agent.log`，不上传日志内容。
- 源码仓库不提交授权文件、账号资料、签名私钥、真实商业脚本和 JSONL 运行数据。
