# v1.27.0 发布说明

## 版本定位
`v1.27.0` 是客户端体验与脚本故障恢复版本，重点修复日志窗口乱码、DM 自动初始化、Lua 报错后无法远程停止，以及控制台操作分散的问题。

## 已包含
- Client 日志窗口：
  - 强制 UTF-8 读取日志，修复中文乱码。
  - 将 13 位毫秒时间戳显示为本地时间。
  - 使用 Client 图标和更清晰的窗口布局。
- Lua / DM：
  - 需要 COM 的 DM 函数首次调用时自动初始化 DmBridge。
  - Web Admin DM/Lua 示例移除强制手写 `dm.init("")`。
  - 脚本推送面板新增 DM 示例脚本。
- Client monitor：
  - Lua 执行失败后仍继续同步 Server 消息和命令。
  - 远程 `script.stop` 不再被持续失败的 Lua 脚本阻断。
- Web Admin：
  - 远程操作页改为左侧目标栏、右侧操作区。
  - 顶部状态栏固定，宽屏布局更集中。
  - “打开日志”改为打开客户端日志 UI。
- 发布包：
  - 使用用户提供的 `Client.ico` 和 `Server.ico`。
  - 发布包内 Client/Server 根 EXE 与 core EXE 写入对应图标资源。

- `cargo fmt --all --check`：通过。
- `cargo test --workspace`：通过。

- `tools/package-release.ps1`：通过，三类 Windows zip 已生成。
- Client 分包 smoke：通过，`bin/client-agent-core.exe --run-once` 输出 `release_version = v1.27.0`、`arch = x86`、Lua 启用、脚本安全关闭和 `dm.access`。
- Server 分包 health smoke：通过，临时端口 `/health` 返回 `{"status":"ok"}`。
- Server 分包命令队列 smoke：通过，`script.deploy_bundle` 可写入并取出命令队列。
- 包内容检查：通过，总包和 Client 分包包含 `DmBridge.dll`、`dm.dll`、`RegDll.dll`；Server 分包不包含这三个 DM 文件。

## 边界
- `script.stop` 阻止后续轮次继续执行 Lua，不强杀已经进入执行中的 Lua 调用。
- Server 桌面控制台仍依赖 Edge App 模式；完全原生 WebView2 壳后续单独实现。
- EXE 图标写入只作用于 release package 副本，不修改 `target/release` 原始编译产物。
