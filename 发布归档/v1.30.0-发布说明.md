# v1.30.0 发布说明

`v1.30.0` 是原生桌面控制台入口修正版本，重点解决用户实测中 Server 控制台仍以 Edge App 模式打开的问题。

## 已包含
- 新增 Rust `desktop-console` 模块，发布包内入口为 `WoW-Desktop.exe`。
- Server 托盘默认启动 `WoW-Desktop.exe --url http://127.0.0.1:18080`，不再调用 `msedge.exe --app`。
- 托盘仍保留“浏览器打开 Web 管理页”作为排错备用入口。
- 完整包和 Server 分包根目录均包含 `WoW-Desktop.exe`，并写入 Server 图标。
- 打包脚本补齐 Windows PowerShell 5.1 相对路径兼容函数，避免旧 PowerShell 缺少 `Path.GetRelativePath` 导致打包失败。
- 文档明确说明：桌面壳使用 Windows WebView2 Runtime，但不是 Edge 浏览器窗口。

## 验证结果
- `cargo fmt --all --check`：通过。
- `cargo test -p desktop-console`：通过。
- `cargo test -p management-server tray`：通过，托盘脚本不包含 `msedge.exe` 或 `--app=$serverUrl`。
- `cargo test --workspace`：通过，Client 55 项、desktop-console 4 项、Server 53 项、shared-types 11 项、launcher 3 项测试全部通过。
- `cargo clippy --workspace -- -D warnings`：通过。
- `npm run build`：通过，Web Admin 版本为 `1.30.0`。
- `tools/package-release.ps1`：首轮构建通过，安全检查因 Windows PowerShell 5.1 缺少 `Path.GetRelativePath` 暴露兼容问题；已修复后使用 `-SkipBuild` 成功生成三类 Windows zip。
- Server 分包 smoke：通过，`/health` 返回 HTTP 200，首页返回 HTTP 200。
- Client 分包 smoke：通过，`bin/client-agent-core.exe --run-once` 输出 `release_version = v1.30.0`、`arch = x86`、Lua 启用、脚本安全关闭和 `dm.access`。
- 桌面壳错误日志 smoke：通过，非法 URL 会写入 `logs/desktop-console-error.log`。
