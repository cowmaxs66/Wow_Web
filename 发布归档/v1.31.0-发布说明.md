# v1.31.0 发布说明

`v1.31.0` 是 DM API 测试与上线准入版本，重点解决目标窗口未打开时被误判为接口失败的问题，并补齐上线前可重复执行的 DM/Lua 自检脚本。

## 已包含
- `dm.find_window` 找不到窗口时返回 `0`，适合业务脚本先探测再决定是否绑定。
- 新增 `dm.find_window_try`，返回 `{ ok, hwnd, error }`，便于脚本在 UI 和回执里展示清晰结果。
- `dm.find_window_required` 保留强失败语义，只用于必须找到窗口才能继续的脚本。
- 新增 `scripts/dm_api_selftest.lua`，覆盖 DM 初始化、版本、取色、颜色等待、窗口探测、错误查询、休眠和时间等低风险接口。
- 新增 `scripts/dm_window_smoke.lua`，覆盖 `World of Warcraft` 窗口查找、绑定、取色和解绑；未找到窗口时返回 `window=not_found`，不让 Client monitor 反复报错。
- Web Admin 脚本推送面板新增 `API 自检` 和 `窗口 smoke` 示例入口。
- Lua 接口表和 DM API 上线准入说明已补齐测试层级、接口覆盖范围和排错方式。

## 测试顺序
1. 在管理端对目标 Client 下发 `API 自检`，确认基础 DM/Lua 链路可用。
2. 打开目标游戏窗口后下发 `窗口 smoke`，确认窗口标题、绑定模式、取色和解绑正常。
3. 业务脚本先使用 `dm.find_window` 或 `dm.find_window_try`，确认窗口存在后再调用 `dm.safe_bind_window`。
4. `move_to`、`left_click` 等真实输入接口不放入默认自检，必须在人工确认目标窗口后单独测试。

## 安全边界
- 总包和 Client 分包继续包含 `DmBridge.dll`、`dm.dll` 和 `RegDll.dll`。
- Server 分包不包含 DM 运行文件。
- 未归档授权文件、CHM/CHW 文档、真实账号资料、商业脚本、签名私钥和 JSONL 运行数据。
- `dm_api_selftest.lua` 不点击、不移动鼠标、不绑定真实业务窗口；它不能替代真实业务脚本实机测试。
