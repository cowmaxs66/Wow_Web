# v1.31.1 发布说明

`v1.31.1` 是绑定探测热修复版本，重点处理用户实测微信窗口 `BindWindow exception / EAccessViolation` 的排查体验。

## 已包含
- 新增 `scripts/dm_bind_probe.lua`，默认探测微信窗口 `Qt51514QWindowIcon / 微信`。
- 绑定探测会逐个尝试 `normal/normal/normal`、`normal/windows/windows`、`gdi/windows/windows`、`gdi2/windows/windows`。
- 每个组合都会写入脚本日志和命令回执，格式为 `ok#` 或 `fail#`，便于判断目标窗口是否可绑定。
- Web Admin 脚本推送面板新增 `绑定探测` 按钮，不需要手动粘贴整段 Lua。
- 脚本目录说明、Lua 接口表、DM API 上线准入说明和部署指南已补充 `BindWindow exception` 处理方式。

## 问题解释
用户日志中的 `hwnd=5772832` 说明窗口已经找到；失败点是大漠 COM 在绑定微信 Qt 窗口时内部抛出 `EAccessViolation`。框架已经捕获异常并返回错误，因此 Client 不会崩溃。这个错误通常表示目标窗口和当前绑定组合不兼容，不是 Lua 语法问题。

## 测试建议
1. 在 Web 管理端选中目标 Client。
2. 进入脚本推送面板，点击 `绑定探测`。
3. 推送并执行后查看命令回执：
   - 出现 `ok#`：业务脚本使用该组合。
   - 全部是 `fail#`：该窗口当前不适合绑定，不要继续执行点击类脚本。

## 安全边界
- `dm_bind_probe.lua` 不点击、不移动鼠标，只尝试绑定、取色和解绑。
- 总包和 Client 分包继续包含 `DmBridge.dll`、`dm.dll` 和 `RegDll.dll`。
- Server 分包不包含 DM 运行文件。
- 未归档授权文件、CHM/CHW 文档、真实账号资料、商业脚本、签名私钥和 JSONL 运行数据。
