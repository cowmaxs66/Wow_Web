# v1.32.0 发布说明

`v1.32.0` 是 WebSocket 实时通讯版本，重点解决多机器管理下命令下发和管理端刷新延迟问题。

## 已包含
- 新增 `/ws/client/{client_id}`，Client 可常驻连接 Management Server。
- 新增 `/api/client/ws/{client_id}` 兼容入口。
- 新增 `/ws/admin`，Web Admin 可接收 Client 状态、命令、回执和消息事件。
- Server 命令仍先写入原队列，再通过 WebSocket 推送给在线 Client。
- Client monitor 与 WebSocket worker 共用命令去重集合，避免 HTTP 和 WS 同时收到命令时重复执行。
- Client 执行远程命令后优先通过 WebSocket 上报回执。
- Server 收到命令回执后，会按 `command_id` 清理待命令队列。
- Web Admin 收到实时事件后做 300ms 防抖刷新，仍保留原分页、筛选和离线判定。

## 使用说明
- Client 分包正常双击 `client-agent.exe` 即可，内部 monitor 会自动启动 WebSocket worker。
- Server 分包正常双击 `management-server.exe` 或打开 `WoW-Desktop.exe` 即可。
- Web Admin 顶部显示“实时通道”代表 `/ws/admin` 已连接；显示“HTTP 保底”时仍可手动刷新并继续使用旧 API。
- 如果 WebSocket 断开，Client 仍会通过 `/api/client/sync` 拉取命令。

## 安全边界
- WebSocket 当前不包含登录鉴权和 TLS，只适合内部测试或受控内网。
- 生产联网前必须补登录、角色权限、操作者审计、TLS、连接限流和防重放。
- 总包和 Client 分包继续包含 `DmBridge.dll`、`dm.dll` 和 `RegDll.dll`。
- Server 分包不包含 DM 运行文件。
- 未归档授权文件、CHM/CHW 文档、真实账号资料、商业脚本、签名私钥和 JSONL 运行数据。
