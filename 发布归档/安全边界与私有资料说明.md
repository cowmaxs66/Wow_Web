# 安全边界与私有资料说明

## 仓库边界
当前 GitHub 仓库保存框架源码、示例配置、技术设计、验证记录和发布资料。以下内容禁止提交：
- `.env`、真实账号、真实脚本配置、商业脚本和业务数据。
- 签名私钥、授权文件、注册码、机器码绑定资料。
- `dm.dll`、`RegDll.dll`、CHM/CHW、RAR、DAT 和大漠私有资料。
- 本地构建产物：`target/`、`node_modules/`、`dist/`、DLL、EXE、PDB、DCU。

## 当前安全能力
- Client Agent 执行 Lua 前会校验 manifest。
- manifest 签名使用 Ed25519 公钥验证。
- Lua 文件内容使用 SHA-256 校验。
- Lua API 按权限注册，未授权 API 不进入 Lua globals。
- DmBridge 通过 C ABI 暴露最小接口，Rust/Lua 不直接接触大漠 COM 指针。

## 当前未解决的生产安全项
| 项目 | 当前状态 | 生产前要求 |
|------|----------|------------|
| 鉴权 | 未实现 | Server 和 Web Admin 必须增加身份认证 |
| TLS | 未实现 | 远程访问必须使用 HTTPS/WSS |
| CORS | 开发期宽松 | 生产必须改为来源白名单 |
| 脚本下发 | 未实现 | 必须先设计签名、审批、审计和回滚 |
| 私钥管理 | 未产品化 | 私钥必须脱离仓库保存，支持轮换与吊销 |
| 数据持久化 | 未实现 | 引入数据库前必须设计 schema、备份和访问控制 |

## 大漠接口边界
第一轮只保留窗口、后台绑定、基本、算法、图色、键鼠相关能力的最小链路。以下高风险方向不进入 v1.0.0：
- 内存读写。
- 汇编或注入。
- 防护盾绕过。
- 变速和进程级控制。
- 未经验证的全量接口暴露。

## 交接要求
后续任何扩展只允许增量接入，不能绕过当前安全门。涉及远程执行、脚本下发、账号权限、真实游戏配置或大漠授权资料时，必须先更新风险记录和技术设计，再进入实现。
