# 产品化控制中心与安装体验设计

## 目标
把当前“多个 exe + 维护脚本”的运行方式收敛为普通用户能理解的本机控制中心，让用户优先通过 `WoW-Manager.exe` 完成安装、启动、查看日志和卸载。

## 影响评估
- 不改变 Server、Client、Web Admin 的协议和运行逻辑。
- 不改变 `management-server.exe` 和 `client-agent.exe` 的正式启动入口。
- `WoW-Manager.exe` 从直接执行安装脚本升级为控制中心入口；安装脚本仍保留，由控制中心调用。
- 当前用户安装目录仍为 `%LOCALAPPDATA%\WoWFramework`。

## 控制中心能力
控制中心由 `installer/manager-current-user.ps1` 提供 WinForms 界面，入口为 `WoW-Manager.exe`。

第一版按钮：
- 安装 / 修复
- 启动 Server
- 启动 Client
- 打开 Web 管理页
- 打开日志目录
- 打开程序目录
- 刷新状态
- 卸载程序
- 关闭窗口

## 安装体验
- 安装/修复会复制当前发布包到 `%LOCALAPPDATA%\WoWFramework`。
- 控制中心支持从已安装目录打开，避免修复安装时误删自身。
- 桌面和开始菜单会创建：
  - `WoW Framework`
  - `WoW Server`
  - `WoW Client`
  - `Remove WoW Framework`
- 快捷方式绑定 Server/Client 图标。

## 非目标
- 本阶段不生成 MSI/MSIX。
- 本阶段不写入系统级安装目录。
- 本阶段不自动安装 Server Windows Service。
- 本阶段不处理代码签名证书、自动更新签名验证和企业部署策略。
