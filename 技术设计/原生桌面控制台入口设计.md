# 原生桌面控制台入口设计

## 目标
P38 解决用户实测中“Server 控制台仍是 Edge 网页窗口”的问题。目标是让 Server 托盘默认打开发布包内的独立桌面程序 `WoW-Desktop.exe`，而不是调用 `msedge.exe --app`。

P38-H01 追加修正：用户直接双击 `WoW-Desktop.exe` 时，桌面壳必须先确认本机 Server 是否可连接；如果没有响应，应自动启动同包内 `bin/management-server-core.exe --no-open-browser`，不能把 WebView2 默认连接失败页直接展示给用户。

## 设计边界
- 不重写 Management Server、Web Admin、Client Agent 和远程命令协议。
- 不引入 Tauri 工程模板，避免一次性改动过大。
- 新增 Rust `desktop-console` 模块，只负责桌面窗口和 URL 加载。
- Windows 上使用 WebView2 Runtime 渲染页面，这是 WRY/Tauri/Clash Verge 同类桌面壳的常规方式；它不是 Edge 浏览器窗口。
- 托盘保留“浏览器打开 Web 管理页”作为排错备用入口。

## 启动链路
```text
management-server.exe
  -> Server 托盘脚本
  -> 启动 bin/management-server-core.exe --no-open-browser
  -> 打开 WoW-Desktop.exe --url http://127.0.0.1:18080
  -> 桌面窗口加载内嵌 Web Admin
```

## 模块职责
| 模块 | 职责 |
|------|------|
| `实现模块/desktop-console` | 独立桌面壳，解析 URL、创建窗口、写入启动错误日志 |
| `实现模块/management-server/src/tray.rs` | 托盘默认启动 `WoW-Desktop.exe`，缺失时才回退浏览器 |
| `tools/package-release.ps1` | 将 `desktop-console.exe` 复制为发布包根目录 `WoW-Desktop.exe` 并写入图标 |

## 异常处理
- `WoW-Desktop.exe` 启动失败时会写入 `logs/desktop-console-error.log`。
- `WoW-Desktop.exe` 连接本机 Server 失败时，会尝试启动同包内 `bin/management-server-core.exe`，标准输出和错误会写入 `logs/management-server.log` 与 `logs/management-server-error.log`。
- 如果发布包中缺少 `WoW-Desktop.exe`，托盘会弹出提示并使用浏览器打开 Web 管理页。
- 如果同包内缺少 `bin/management-server-core.exe`，桌面壳会显示本地中文错误页，不再直接显示 WebView2 默认连接失败页。
- 如果目标机器缺少 WebView2 Runtime，桌面壳会启动失败并写入错误日志；这类机器需要安装 Microsoft Edge WebView2 Runtime。

## 验证标准
- `cargo test -p desktop-console` 通过。
- `cargo test -p management-server tray` 通过，且生成的托盘脚本不包含 `msedge.exe` 或 `--app=$serverUrl`。
- 发布包中的完整包和 Server 分包包含 `WoW-Desktop.exe`。
- 发布包 `RUNNING.md` 明确说明桌面控制台不是 Edge 浏览器窗口。
