# 双击正式入口设计

## 目标
P14 将发布包入口从“命令参数驱动”调整为“常规 exe 双击可用”。

## 入口规则
| 程序 | 无参数行为 | 维护参数 |
|------|------------|----------|
| `management-server.exe` | 启动 Management Server 并打开 Web 管理页 | `--no-open-browser` / `--api-only` |
| `client-agent.exe` | 启动托盘常驻 UI，并由托盘拉起 monitor | `--run-once` / `--monitor` |

## Client 行为
- 双击 `client-agent.exe` 等同 `--tray`。
- 托盘启动后自动拉起 monitor 子进程。
- 开机启动写入当前用户 HKCU Run 时只写 `client-agent.exe`，不再写 `--monitor`。
- `--run-once` 保留旧的单次 JSON 输出，用于测试、脚本和维护。

## Server 行为
- 双击 `management-server.exe` 会打开默认浏览器访问本机 Web Admin。
- 自动化或服务化场景使用 `--no-open-browser`，避免弹出浏览器。
- `--open-browser` 保留兼容旧脚本。

## 边界
- 本阶段不引入安装器。
- 本阶段不重构 Service、托盘、设置窗口、更新器和远程命令。
- 本阶段不把内部维护参数展示为普通用户主入口。
