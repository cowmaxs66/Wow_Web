# 命令执行回执与审计设计

## 阶段定位
P24 面向远程操作闭环，解决 P13-P23 期间“Server 只能确认命令写入队列，不能确认 Client 是否执行”的问题。

本阶段只做最小兼容扩展：
- Server 增加命令执行回执 API。
- Client monitor 执行远程白名单命令后回传成功或失败摘要。
- Web Admin 在远程操作页展示当前 Client 最近回执。

## 当前边界
| 项目 | 决策 |
|------|------|
| 协议 | 新增 `ClientCommandReceiptRequest`、`ClientCommandReceipt`、`ClientCommandReceiptList` |
| Server 存储 | 进程内 `HashMap<String, VecDeque<ClientCommandReceipt>>` |
| 每台 Client 上限 | 最近 100 条 |
| Client 上报方式 | 命令执行后 best-effort POST |
| Web 展示范围 | 只展示当前选中的单台 Client，批量下发不混合展示 |

## 数据流
```text
Web Admin 下发命令
  -> Management Server 写入 Client 命令队列
  -> Client monitor 轮询并执行白名单命令
  -> Client monitor POST 执行回执
  -> Management Server 保存最近回执
  -> Web Admin 查询并展示最近执行结果
```

## API
| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/client/command-receipts/{client_id}` | Client 上报命令执行回执 |
| `GET` | `/api/client/command-receipts/{client_id}` | Web Admin 查询最近回执 |

## 回执字段
| 字段 | 说明 |
|------|------|
| `command_id` | Server 生成的命令 ID |
| `command_type` | 白名单命令类型 |
| `success` | Client 本机执行是否成功 |
| `summary` | 执行摘要，最长 2000 字符 |

## 安全边界
- 回执接口只接受已有白名单命令类型，拒绝 `shell.exec` 等任意命令。
- Client 上报回执失败时只写本机日志，不阻断 monitor。
- Server 当前不持久化回执，重启后丢失。
- 当前仍未包含登录鉴权、操作员身份、审批、持久化审计和防重放。

## 后续扩展
- 将命令、回执和操作员身份写入数据库。
- 增加 Server 登录、角色权限和审计查询。
- 对远程更新和脚本执行增加确认窗口、风险等级和回滚记录。
