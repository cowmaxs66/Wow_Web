## 范围结论
用户明确需要以下功能域：

- 窗口
- 后台设置
- 基本设置
- 算法
- 图色
- 键鼠

其他功能域先不纳入第一版桥接，包括：内存、汇编、防护盾、Foobar、答题、文件、文字识别、系统、杂项。

## 设计原则
1. Rust 不直接调用大漠 COM，先通过 Delphi `DmBridge.dll` 包一层稳定 C ABI。
2. Lua 不直接暴露大漠原始全量接口，只暴露经过筛选的高层 API。
3. 大漠 COM 调用固定进入单一 STA 工作线程，避免多线程直接调用 COM。
4. 第一版只做自动化所需的窗口、绑定、图色、键鼠闭环。
5. 所有授权码、注册码、真实脚本和真实配置都禁止进入 GitHub。
6. 内存读写、汇编、防护盾、变速、降 CPU 等高风险接口不进入第一版。

## 明确延后或排除
| 接口或功能域 | 策略 | 原因 |
|--------------|------|------|
| 内存读写 | 排除第一版 | 偏进程内存操作，高风险且不属于图色闭环 |
| 汇编 | 排除第一版 | 与当前目标无关，风险高 |
| 防护盾 | 排除第一版 | 容易走向对抗环境，不进入基础框架 |
| `HackSpeed` | 排除第一版 | 变速能力风险高 |
| `DownCpu` | 排除第一版 | 非核心自动化能力，可能影响系统稳定 |
| `EnableSpeedDx` | 延后 | 仅在明确需要 DirectX 加速时评估 |
| Foobar | 延后 | 调试绘制能力，不是第一闭环 |
| 文字识别 | 延后 | 用户本轮未要求 OCR |
| 文件/系统/杂项 | 延后 | 不属于大漠核心图色键鼠闭环 |

## C ABI 设计方向
Delphi Bridge 不直接导出全部大漠原始函数。第一版导出稳定、少量、强约束的 C ABI：

- 整数返回值：`1` 表示成功，`0` 或负数表示失败。
- 坐标返回值：通过 `out_x/out_y` 输出。
- 字符串输入：优先使用 UTF-16 `PWideChar`，避免中文路径和图片名乱码。
- 字符串输出：使用调用方提供的 UTF-16 buffer，避免跨 DLL 分配释放内存。
- 线程模型：Bridge 内部维护单一 COM STA 调用线程，Rust 侧不得多线程直接打 COM。

## Lua API 暴露方向
Lua 层使用稳定高层接口，不直接使用大漠原始 Pascal/COM 命名：

```lua
dm.set_path("assets")
dm.window.find(nil, "窗口标题")
dm.bind(hwnd, "dx", "windows", "windows", 0)
dm.find_pic(0, 0, 1280, 720, "ok.bmp", "000000", 0.9, 0)
dm.move_to(x, y)
dm.left_click()
dm.unbind()
```
