# 客户端体验与脚本故障恢复设计

## 目标
修复 v1.26.0 实机测试暴露的三个问题：日志窗口中文乱码、Lua 脚本失败后远程停止命令无法及时生效、热推 DM 脚本首次调用时报 `DmBridge is not initialized`。同时优化 Web Admin 远程操作页，让目标选择、脚本推送、命令下发和回执查看集中在同一工作台。

## 设计决策
- DmBridge 在 Lua 首次调用需要 COM 的函数时自动初始化，保留 `dm.init` 给高级脚本显式指定路径。
- Client monitor 即使本轮 Lua 执行失败，也必须继续向 Server 同步状态、消息和命令。
- 日志文件仍保留毫秒时间戳原始格式，日志窗口负责转换为可读本地时间，避免破坏旧日志兼容。
- Web Admin 远程操作页采用左侧目标栏、右侧操作区的控制台布局，减少来回滚动。
- 发布包内 EXE 图标由打包脚本写入 PE 资源，不依赖外部 Resource Hacker。

## 影响范围
- `client-agent`：Lua DM 生命周期、monitor 同步顺序、日志窗口、远程打开日志命令。
- `web-admin`：全局控制台布局、远程操作页、脚本推送面板、DM/Lua 示例。
- `tools/package-release.ps1`：发布包 EXE 图标写入。
- `assets/icons`：替换为用户提供的 Client/Server 图标。

## 边界
- 本阶段不引入 WebView2 原生壳，Server 桌面控制台仍使用 Edge App 模式。
- 本阶段不实现 Lua 强杀；`script.stop` 通过跳过下一轮脚本执行生效。
- 本阶段不增加数据库或登录鉴权。
