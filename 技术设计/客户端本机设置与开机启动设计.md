# 客户端本机设置与开机启动设计

## 阶段定位
| 项目 | 内容 |
|------|------|
| 阶段 | P12 |
| 目标版本 | v1.6.0 |
| 范围 | Client 本机设置入口、当前用户开机启动、Web 向导命令生成 |
| 不包含 | Windows Service、自动更新器、完整托盘右键菜单、原生设置窗口 |

## 设计目标
- Client 可查询、启用、停用当前用户开机启动。
- 开机启动默认运行 `client-agent.exe --monitor`。
- 开机启动不写入 Server 密钥、大漠授权、账号资料或私有脚本路径。
- Web Admin 只生成命令，不直接修改本机系统设置。
- 工具脚本和 exe 命令保持同一套能力。

## 模块拆分
| 模块 | 职责 |
|------|------|
| `client-agent/src/startup.rs` | Windows HKCU Run 注册项读写与状态摘要 |
| `client-agent/src/cli.rs` | 解析 `--startup-status`、`--enable-startup`、`--disable-startup` |
| `client-agent/src/main.rs` | 将 CLI 命令路由到 startup 模块 |
| `tools/start-client.ps1` | 为脚本入口补充开机启动开关 |
| `web-admin/useSetupWizard.ts` | 生成 Client monitor 与开机启动命令 |
| `SetupWizardPanel.vue` | 展示查询、启用、停用开机启动命令 |

## 开机启动实现
| 项目 | 内容 |
|------|------|
| 注册表位置 | `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` |
| 注册项名称 | `WoW Client Agent` |
| 注册命令 | `"当前 client-agent.exe 路径" --monitor` |
| 权限要求 | 当前用户权限，不需要管理员权限 |
| 停用方式 | 删除同名 HKCU Run 注册项 |

## 边界说明
- 当前版本只管理当前用户启动项，不写入 `HKLM`。
- 启动项只记录 exe 路径和 `--monitor`，Server 地址、大漠桥接路径和脚本配置继续来自本机配置文件或环境变量。
- 如果用户移动发布包目录，需要重新执行 `--enable-startup`，让注册项指向新 exe 路径。
- Windows Service 会改变权限、工作目录、桌面交互和大漠接入模型，后续阶段单独设计。

## 验证方式
- `cargo test -p client-agent`：验证 CLI 解析、注册表输出解析和启动命令格式。
- `client-agent.exe --startup-status`：只读查询当前用户启动项。
- `tools/start-client.ps1 -StartupStatus`：验证脚本入口可调用。
- `npm run build`：验证 Web 向导命令生成不会破坏前端构建。
