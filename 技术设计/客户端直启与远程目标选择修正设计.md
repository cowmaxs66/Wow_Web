# 客户端直启与远程目标选择修正设计

## 阶段范围
P19 / v1.13.0。

## 问题边界
- Client 分包根目录 `client-agent.exe` 双击后应直接进入正式托盘运行，并默认向本机 Server 上报。
- 托盘和通知脚本不应弹出黑色控制台窗口。
- Client 正常退出或托盘停止 monitor 后，Server 应收到 `online=false` 状态。
- Web Admin 远程操作应能明确选择单台 Client 或全部已上报 Client。

## 影响评估
| 项目 | 结论 |
|------|------|
| 是否影响旧结构 | 否 |
| 是否影响已完成任务 | 否 |
| 是否需要兼容处理 | 是，保留所有维护命令和现有 Server API |

## 实现方案
- 发布包生成阶段写入正式 Client 配置：`server.enabled=true`，并把 DmBridge 路径修正为包内相对路径。
- Client 托盘宿主和通知宿主使用隐藏 PowerShell 进程，并保留 GUI launcher。
- Client 增加内部 `--report-offline` 维护命令，托盘停止 monitor 时补发离线状态；Service 停止仍走 monitor 的优雅退出。
- Server 心跳兜底窗口由 60 秒收敛为 30 秒，处理非正常杀进程场景。
- Web Admin 远程操作面板新增目标选择框，支持单台和全部已上报 Client，发送时逐个写入命令/消息队列。

## 不纳入本阶段
- 远程命令执行回执。
- 登录鉴权、审计、权限分级。
- MSI/MSIX 安装器。
- 发布包独立签名校验。

## 验证标准
- 包内 `config/client-agent.toml` 默认为 `server.enabled=true`。
- Client 分包不依赖 `tools/start-client.cmd` 也能上报状态。
- 正常退出/托盘停止后 Server 最新状态为离线。
- Web Admin 构建通过，并能按单个 Client 或全部 Client 写入队列。
- 三类发布包仍通过敏感文件检查。
