# 服务端上线日志与分包设计

## 目标
P18 解决两个正式使用问题：
- Client 启动并上报后，Server 控制台必须能看到上线、刷新、离线等状态日志。
- 发布包除保留总包外，同时输出 Server 分包和 Client 分包，便于分别部署。

## 影响评估
- 是否影响旧结构：否。总包 `WoW_Framework_*` 继续保留。
- 是否影响已完成任务：否。P15 GUI launcher、P16 updater、P17 远程更新命令继续兼容。
- 是否需要兼容处理：是。Client updater 仍默认匹配总包，不改成分包，避免误下载 Server 分包。

## Server 控制台日志
Server 接收 `POST /api/client/status` 后，按上一条状态判断事件类型：

| 条件 | 控制台事件 |
|------|------------|
| 第一次看到该 Client | `Client 上线` |
| 上一次已离线或超时离线，本次在线 | `Client 上线` |
| 上一次在线，本次仍在线 | `Client 状态刷新` |
| 本次上报 offline | `Client 离线` |

日志字段：
- `client_id`
- `online`
- `script`
- `release_version`
- `message_id`

说明：根目录 `management-server.exe` 是 GUI launcher，本身不显示控制台；要查看控制台日志，运行分包或总包中的 `bin/management-server-core.exe --no-open-browser`。

## 分包输出
`tools/package-release.ps1` 现在输出三类 zip：

| 包 | 文件名 | 用途 |
|----|--------|------|
| 总包 | `WoW_Framework_vX.Y.Z_windows.zip` | 完整测试包、当前用户安装器、自动更新兼容包 |
| Server 分包 | `WoW_Server_vX.Y.Z_windows.zip` | 只部署 Management Server 和 Web Admin |
| Client 分包 | `WoW_Client_vX.Y.Z_windows.zip` | 只部署 Client Agent、DmBridge、配置和脚本 |

## 包内容边界
### Server 分包
- `management-server.exe`
- `bin/management-server-core.exe`
- `tools/common.ps1`
- `tools/start-server.ps1`
- `README.md`
- `VERSION`
- `RUNNING.md`

### Client 分包
- `client-agent.exe`
- `bin/client-agent-core.exe`
- `bin/client-agent-x64-core.exe`
- `dm-bridge/Win32/DmBridge.dll`
- `config/client-agent.toml`
- `scripts/`
- `tools/common.ps1`
- `tools/start-client.ps1`
- `README.md`
- `VERSION`
- `RUNNING.md`

## 风险与边界
- Server GUI launcher 仍不显示控制台，这是无控制台入口的预期行为。
- Client updater 继续下载 `WoW_Framework` 总包，避免匹配到 Server/Client 分包造成半更新。
- 分包不包含大漠官方 `dm.dll`、`RegDll.dll`、授权文件、真实账号和 JSONL 运行数据。
- Server 分包不包含 Client 配置、脚本和 DmBridge。
- Client 分包不包含 Management Server 可执行文件。

## 验证方式
- `cargo fmt --all --check`
- `cargo test --workspace`
- `cargo clippy --workspace -- -D warnings`
- `npm run build`
- `tools/package-release.ps1`
- 检查三类 zip 均生成，并分别通过敏感文件检查。
- 包内 Server 状态上报烟测，确认控制台日志包含 `Client 上线`。
