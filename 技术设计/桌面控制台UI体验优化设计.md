# 桌面控制台 UI 体验优化设计

## 阶段定位
P37 / v1.29.0 只优化现有 Web Admin/桌面控制台的信息架构和交互体验，不改变 Management Server、Client Agent、远程命令和 Lua 执行链路。

## 用户视角问题
- 控制台之前更像后台页面，用户需要在多个页面之间寻找 Client、脚本、设置和日志。
- 总览页先展示统计，真正可操作的 Client 列表不够靠前。
- Server 审计和命令回执没有独立导航入口，排查脚本推送失败时路径不直观。
- 移动或窄窗口下导航容易挤压，影响桌面 App 模式下的缩放体验。

## 设计原则
- 第一屏先回答“Server 是否可用、当前选中哪台 Client、下一步能做什么”。
- 目标选择、脚本、设置、日志都围绕选中 Client 展开。
- 保留已有组件和 API，避免为了 UI 重构破坏远程命令闭环。
- 控制台要像桌面工具，不做营销式首页，不增加无关装饰。

## 实现范围
| 模块 | 调整 |
|------|------|
| AppShell | 侧栏改为深色桌面工具风格，增加日志导航和 Server 状态摘要 |
| App.vue | 增加顶部当前目标卡片、独立日志页、总览页 Client 列表优先 |
| ClientTable | 修正加载中、无 Client、筛选无结果三种空状态 |
| 样式 tokens | 增加侧栏主题变量，统一输入控件 accent 色 |
| QA | 增加桌面和移动视口浏览器验证截图 |

## 交互流程
```text
打开控制台
  -> 查看 Server 状态和当前目标
  -> 在 Client 列表选择机器
  -> 进入脚本、远程操作或设置
  -> 到日志与回执页查看结果和错误
```

## 兼容边界
- 不新增后端 API。
- 不改变命令队列、回执、审计和 Client 设置协议。
- 不移除旧页面能力，只调整入口和布局顺序。
- Server 托盘继续使用 Edge App 模式打开桌面控制台。

## QA 标准
- `npm run build` 必须通过。
- 1280px 桌面视口无横向溢出。
- 390px 移动视口无横向溢出，导航文字不得竖排。
- 日志页必须能从侧栏进入，并显示 Server 审计和命令回执入口。
- 浏览器控制台错误数为 0。
