# 正式运行基础设计

## 阶段定位
| 项目 | 内容 |
|------|------|
| 阶段 | P13 |
| 目标版本 | v1.7.0 |
| 范围 | Windows Service、托盘常驻、原生设置窗口、自动更新检查/下载、Server 远程下发 Client 本机操作 |

## 目标
- Client 不再依赖用户手动执行多条测试命令。
- 后台运行由 Windows Service 或托盘 monitor 管理。
- 常用操作集中到托盘右键菜单和 Web 管理端。
- 本机设置可通过原生窗口编辑。
- 更新器能检查和下载 GitHub 最新发布包。
- Server 可下发白名单 Client 本机操作。

## 模块拆分
| 模块 | 职责 |
|------|------|
| `client-agent/src/service_runtime.rs` | Windows Service 运行入口与 `sc.exe` 管理命令 |
| `client-agent/src/tray.rs` | WinForms 托盘常驻、右键菜单、monitor 子进程控制 |
| `client-agent/src/settings_window.rs` | WinForms 本机配置编辑窗口 |
| `client-agent/src/updater.rs` | GitHub Release 检查与下载 |
| `client-agent/src/remote_command.rs` | Server 白名单命令执行分发 |
| `management-server/src/state.rs` | Client command 内存队列 |
| `management-server/src/app.rs` | Client command API |
| `web-admin/src/components/ClientDetail.vue` | Web 远程本机操作区 |

## Service 边界
- `--service-run` 是真正的 Windows Service 入口，使用 `windows-service` 注册 SCM handler。
- `--service-install/start/stop/status/uninstall` 负责管理 Service。
- Service 只跑 monitor，不打开窗口，不显示托盘，避免 Session 0 交互问题。
- Service 安装、启动和停止通常需要管理员权限。

## 托盘边界
- `--tray` 启动 WinForms NotifyIcon。
- 右键菜单包含 monitor、设置窗口、日志、开机启动、Service、更新和退出。
- 托盘运行在当前用户 Session，可打开窗口和通知。
- 托盘默认拉起 monitor 子进程；退出托盘时停止该子进程。

## 设置窗口边界
- `--settings-window` 打开 WinForms 配置窗口。
- 当前版本编辑 `client-agent.toml` 原文，保存后 monitor/service 下次刷新读取。
- 后续可以替换为字段化表单，但不影响配置文件格式。

## 更新器边界
- `--update-check` 查询 GitHub latest release。
- `--update-download` 下载最新 `WoW_Framework*.zip` 到 `%LOCALAPPDATA%\WoWFramework\updates`。
- 当前版本不做运行中自替换，避免替换正在运行的 exe 导致损坏。

## Server 远程下发边界
- 新增 `ClientCommand` 队列，与普通 Server 消息分离。
- API：`POST/GET /api/client/commands/{client_id}`。
- Web 只能选择白名单命令，不提供任意命令输入。
- Client monitor 轮询命令并执行白名单动作。
- 当前命令队列仍为内存队列，鉴权、审计、持久化和送达确认后续阶段继续补齐。

## 命令白名单
| 命令 | 作用 |
|------|------|
| `startup.status` | 查询开机启动 |
| `startup.enable` | 启用开机启动 |
| `startup.disable` | 停用开机启动 |
| `service.status` | 查询 Service |
| `service.install` | 安装 Service |
| `service.start` | 启动 Service |
| `service.stop` | 停止 Service |
| `update.check` | 检查更新 |
| `update.download` | 下载更新包 |
| `settings.open` | 打开设置窗口 |
| `log.open` | 打开日志 |
| `tray.open` | 打开托盘 |
