# 脚本管理与安全机制设计

## 目标
P5 建立脚本执行前的最小安全门：

```text
client-agent.toml -> script manifest -> Ed25519 签名校验 -> SHA-256 校验 -> 权限白名单 -> Lua 执行
```

## 本阶段做什么
| 能力 | 说明 |
|------|------|
| manifest 管理 | 每个脚本入口必须有 manifest 描述 |
| 版本记录 | manifest 记录 `script_id` 和 `version` |
| 完整性校验 | Lua 文件内容必须匹配 manifest 中的 `sha256` |
| 签名校验 | manifest 安全关键字段必须通过 Ed25519 签名 |
| 权限白名单 | manifest 请求权限必须被本机配置允许 |
| Lua API 条件注册 | 未授权 API 不注册到 Lua globals |

## 本阶段不做什么
- 不做远程脚本下发。
- 不做 Web 端脚本管理扩展。
- 不做数据库脚本仓库。
- 不提交签名私钥。
- 不做自动签名工具链。
- 不执行真实业务脚本、账号配置或窗口标题采集。

## manifest 格式
```json
{
  "schema_version": 1,
  "script_id": "bootstrap",
  "version": "0.1.0",
  "entry": "bootstrap.lua",
  "sha256": "...",
  "permissions": ["host.log", "config.read"],
  "signature": "..."
}
```

## 签名 payload
签名不直接覆盖 JSON 文本，避免字段顺序和空白造成误判。P5 签名固定覆盖：

```text
wow-framework-script-manifest-v1
schema_version=1
script_id=...
version=...
entry=...
sha256=...
permissions=...
```

`permissions` 在签名前按字典序排序。

## 当前权限
| 权限 | 说明 |
|------|------|
| `host.log` | 允许 Lua 调用 `log()` |
| `config.read` | 允许 Lua 调用 `get_config()` 读取白名单配置 |
| `dm.access` | 允许 Lua 访问 `dm` 大漠高层 API |

## 安全规则
- `script_security.enabled = true` 时必须提供 manifest。
- manifest `script_id` 必须等于 `lua.bootstrap_name`。
- manifest `entry` 必须是相对路径。
- manifest `entry` 必须与 `lua.bootstrap_path` 指向同一脚本。
- manifest 签名必须由配置中的 `trusted_signer_public_key` 验证通过。
- Lua 文件 hash 必须等于 manifest `sha256`。
- manifest 请求权限必须全部包含在 `allowed_permissions` 中。
- 校验失败时不进入 Lua VM。

## 目录排版
| 文件 | 职责 |
|------|------|
| `src/script/mod.rs` | 脚本加载总流程 |
| `src/script/manifest.rs` | manifest 读取与字段校验 |
| `src/script/signature.rs` | Ed25519 签名校验 |
| `src/script/hash.rs` | SHA-256 摘要 |
| `src/script/permissions.rs` | 权限常量与集合 |
| `src/script/error.rs` | 脚本安全错误 |
| `src/lua_host.rs` | 按权限注册 Lua API |
| `scripts/bootstrap.manifest.json` | 开发期 bootstrap manifest |

## 验收标准
- hash 不匹配必须拒绝加载。
- 签名不匹配必须拒绝加载。
- 权限不在白名单必须拒绝加载。
- 未授权 Lua API 不应注册。
- 默认 `cargo run -p client-agent` 仍能运行。
- `cargo clippy --workspace -- -D warnings` 和 `cargo test --workspace` 通过。
