# 远程设置与脚本日志回执设计

## 阶段定位
P36 / v1.28.0 解决两个实机使用问题：
- Client 设置入口必须在控制面板内可直接操作，不要求用户去客户端机器手动改。
- Lua 脚本的 `return` 和 `log()` 必须能随远程命令回执展示，并按类别显示，便于排查 DM 脚本。

## 影响评估
- 不改变 `config.apply` 协议，只复用既有白名单配置补丁。
- 不新增任意命令执行能力，只把已允许的 Client 设置面板前移到远程操作页。
- 不改变命令回执数据结构，仍使用 `summary` 字符串；新增 `[script.result]`、`[script.log]` 等行前缀供 Web 分类展示。

## 实现方案
- Client Lua 宿主在执行脚本时收集 `log()` 文本，保留原有 tracing 写日志行为。
- `script.deploy_bundle`、`script.run_bootstrap` 和 `script.start` 立即执行 Lua 后，把脚本返回值和脚本日志追加到命令回执。
- Web Admin 回执列表解析行前缀，显示为“执行摘要 / 脚本输出 / 脚本日志 / 日志省略 / 错误摘要”。
- Web Admin 远程操作页加入 `ClientConfigApplyPanel`，并提供快捷模板：
  - DM 内部测试
  - 仅状态上报
  - 停止 Lua
  - 安全校验模式

## 边界
- 回执摘要有长度限制，只展示最近一批脚本日志，避免大量日志撑爆 Server 内存回执。
- 远程设置仍不允许修改 `client.id`，避免历史状态、命令队列和回执断裂。
- 模板只填入表单，不会自动下发；用户仍需点击“套用到 Client”确认。
