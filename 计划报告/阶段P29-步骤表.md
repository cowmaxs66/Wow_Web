# 阶段P29-步骤表

## 阶段目标

完成多机器管理资料模型、Client 分组/标签和批量操作安全边界。

## 步骤总表

| 步骤 | 名称 | 主要产物 | 当前状态 | 备注 |
|------|------|----------|----------|------|
| P29-S01 | Client 身份模型 | `ClientIdentityInfo`、`client.display_name/group/tags` | 已完成 | `client.id` 不允许远程修改 |
| P29-S02 | Web 分组/标签展示 | Client 列表、详情、快照分析 | 已完成 | 支持搜索分组与标签 |
| P29-S03 | 远程配置身份补丁 | `ClientIdentityConfigPatch` | 已完成 | 只允许改显示名、分组、标签 |
| P29-S04 | 批量操作安全边界 | Web 批量确认保护 | 已完成 | 全部 Client 下发前必须勾选确认 |

## 完成标准

- Client 状态上报包含显示名、分组和标签。
- Web Admin 能直接展示分组与标签，不依赖 JSON 文本查看。
- 批量消息和批量命令都有显式确认。
- Rust 与 Web 验证通过，并同步项目文档。
