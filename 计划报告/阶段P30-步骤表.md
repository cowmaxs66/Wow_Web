# 阶段P30-步骤表

## 阶段目标

降低多机器轮询请求尖峰，并把 Client monitor 的状态上报、消息拉取、命令拉取合并为一次同步请求。

## 步骤总表

| 步骤 | 名称 | 主要产物 | 当前状态 | 备注 |
|------|------|----------|----------|------|
| P30-S01 | monitor jitter | `CLIENT_AGENT_MONITOR_JITTER_MS` | 已完成 | 默认最大 1500ms |
| P30-S02 | 合并同步协议 | `ClientSyncRequest/ClientSyncResponse` | 已完成 | 共享协议层 |
| P30-S03 | Server sync API | `POST /api/client/sync` | 已完成 | 保存状态后返回消息和命令 |
| P30-S04 | Client sync 优先链路 | `StatusReporter::sync_client` | 已完成 | 失败回退旧轮询链路 |

## 完成标准

- monitor 每轮优先走 `/api/client/sync`。
- sync 失败时能回退旧 API，不阻断旧 Server 使用。
- Server API 测试覆盖 sync 保存状态、返回消息、取出命令。
- 全量验证通过，并生成 v1.22.0 本地编译包。
