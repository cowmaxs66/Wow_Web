# 阶段P31-步骤表

## 阶段目标

补齐多机器管理的 Server 查询能力和第一版长期审计记录，让 Web Admin 在 Client 数量增加时仍能按页、分组和标签查看，并能追踪最近 Server 操作。

## 步骤总表

| 步骤 | 名称 | 主要产物 | 当前状态 | 备注 |
|------|------|----------|----------|------|
| P31-S01 | Client 分页协议 | `ClientStatusPage` | 已完成 | 共享协议层 |
| P31-S02 | Server 分页过滤 API | `GET /api/client/status-page` | 已完成 | 支持分组、标签、在线和搜索 |
| P31-S03 | Server 审计模型 | `ServerAuditEvent` | 已完成 | 消息、命令、回执摘要 |
| P31-S04 | 审计 JSONL 持久化 | `MANAGEMENT_SERVER_AUDIT_PATH` | 已完成 | 未配置时保持内存事件 |
| P31-S05 | Web 分页过滤与审计面板 | `ClientTable.vue`、`AuditPanel.vue` | 已完成 | 保留旧接口回退 |
| P31-S06 | 验证、打包和归档 | `v1.23.0` 三类包 | 已完成 | GitHub Release 已创建 |

## 完成标准

- `GET /api/client/status-page` 能返回分页元数据和当前页 Client。
- 分组、标签、在线状态和关键字过滤可组合使用。
- `GET /api/server/audit` 能返回最近审计事件。
- 配置 `MANAGEMENT_SERVER_AUDIT_PATH` 后审计事件写入 JSONL，Server 重启可回放最近事件。
- Web Admin 可使用 Server 端筛选和分页，并可查看最近审计。
- 全量验证通过，并生成 v1.23.0 本地编译包。
