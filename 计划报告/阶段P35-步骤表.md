# 阶段P35-步骤表

## 阶段目标
修复日志乱码、DM 自动初始化、Lua 失败阻断远程停止命令的问题，并优化 Web Admin 控制台操作体验和发布包 EXE 图标。

## 步骤总表
| 步骤 | 名称 | 主要产物 | 当前状态 | 备注 |
|------|------|----------|----------|------|
| P35-S01 | Client 日志窗口修复 | UTF-8 读取、时间格式化、Client 图标 | 已完成 | 解决中文乱码和时间不可读 |
| P35-S02 | DM 自动初始化 | Lua DM 生命周期修正 | 已完成 | 首次 DM 调用自动 init |
| P35-S03 | Lua 失败后仍同步命令 | monitor 故障恢复链路 | 已完成 | 脚本报错时仍能接收 stop |
| P35-S04 | Web Admin 控制台优化 | 远程操作工作台、脚本示例、顶部布局 | 已完成 | 降低操作分散和滚动成本 |
| P35-S05 | 图标、验证与发布 | EXE 图标写入、三类 zip、GitHub Release | 已完成 | v1.27.0 已发布 |

## 完成标准
- 日志窗口中文显示正常，13 位毫秒时间戳显示为本地可读时间。
- Lua 直接调用 `dm.find_window_required` 不再因未手动 `dm.init` 报初始化错误。
- Lua 脚本失败时，Client monitor 仍能拉取 Server 下发的停止或替换脚本命令。
- 发布包内 Client/Server EXE 能读取对应图标资源。
- Web Admin 远程操作页目标选择、脚本推送、命令下发和回执查看集中展示。
