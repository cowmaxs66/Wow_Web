# 阶段P36-步骤表

## 阶段目标
把 Client 远程设置入口前移到控制面板，并让 Lua 脚本输出和 `log()` 能在管理端按分类查看。

## 步骤总表
| 步骤 | 名称 | 主要产物 | 当前状态 | 备注 |
|------|------|----------|----------|------|
| P36-S01 | Lua 日志回执采集 | `ScriptRunReport.log_messages`、回执行前缀 | 已完成 | 支持 `return` 和 `log()` 上报 |
| P36-S02 | Web 回执分类展示 | 回执分类 UI | 已完成 | 脚本输出、日志、错误分块 |
| P36-S03 | 控制面板远程设置 | 操作页设置面板和快捷模板 | 已完成 | 不再必须去客户端设置 |
| P36-S04 | 验证与归档 | 编译测试、文档、发布包 | 已完成 | v1.28.0 已发布 |

## 完成标准
- 远程推送脚本后，命令回执能显示 Lua `return`。
- Lua `log()` 能在命令回执中按“脚本日志”展示。
- 远程操作页可直接打开 Client 设置面板并套用配置。
- 快捷模板不会自动下发，必须人工点击确认。
- Rust 和 Web Admin 构建验证通过。
