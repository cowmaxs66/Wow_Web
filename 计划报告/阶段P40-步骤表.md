# 阶段P40-步骤表

## 阶段目标
完成 WebSocket 实时通讯通道，让多机器命令下发从纯轮询升级为实时推送，并保留 HTTP 合并同步保底。

## 步骤总表
| 步骤 | 名称 | 主要产物 | 当前状态 | 备注 |
|------|------|----------|----------|------|
| P40-S01 | 定义实时通讯协议 | `ClientRealtimeMessage`、`ServerRealtimeMessage`、`AdminRealtimeMessage` | 已完成 | 不改变原 HTTP DTO |
| P40-S02 | Server 实时连接注册与广播 | `ServerState` realtime registry、`realtime_ws.rs` | 已完成 | 命令仍保留原队列 |
| P40-S03 | Client 实时命令通道 | `client_realtime.rs`、共享命令去重 | 已完成 | WS 失败自动回退 HTTP |
| P40-S04 | Web Admin 实时事件刷新 | `api/realtime.ts`、顶部实时状态 | 已完成 | 事件触发防抖 HTTP 刷新 |
| P40-S05 | 验证与发布归档 | 测试、clippy、Web 构建、WS smoke、发布说明 | 已完成 | 目标版本 v1.32.0 |

## 完成标准
- Client 能连接 `/ws/client/{client_id}` 并接收 Server 推送命令。
- Server 能通过 `/ws/admin` 推送状态、命令和回执事件。
- HTTP `/api/client/sync` 保持可用，断线时继续兜底。
- WS 和 HTTP 同时收到同一命令时不重复执行。
- 全量 Rust 测试、clippy、Web Admin 构建和本机 WS smoke 均通过。
